# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public GitHub issue**.

Report it privately by emailing: **contact@kyns.ai**

Include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

We will acknowledge your report within 48 hours and aim to release a fix within 7 days for critical issues.

## Scope

In scope:
- Authentication and authorization flaws
- Data exposure or privacy leaks
- Injection vulnerabilities (SQL, NoSQL, command injection)
- XSS, CSRF, SSRF
- Rate limiting bypasses
- Dependency vulnerabilities with known exploits

Out of scope:
- Issues requiring physical access to infrastructure
- Social engineering attacks
- Denial of service (without a working proof of concept)

## Disclosure Policy

We follow responsible disclosure. Once a fix is released, you are welcome to publicly disclose the vulnerability. We will credit you in the release notes unless you prefer to remain anonymous.
